# 贡献指南

感谢您考虑为 Docx to PDF Converter 项目做出贡献！我们欢迎任何形式的贡献，无论是报告错误、改进文档，还是提交代码修改。

## 如何贡献

### 报告错误

如果您发现了错误或问题，请在 GitHub 上提交一个 [Issue](https://github.com/用户名/项目名/issues)。请尽量提供详细信息，包括：

- 错误发生的环境（操作系统、Python版本等）
- 复现错误的步骤
- 错误信息或截图
- 预期行为与实际行为的差异

### 提交代码

1. **Fork 项目仓库**
2. **创建您的分支**
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **进行修改**
4. **提交您的更改**
   ```bash
   git commit -m '添加一些令人惊叹的功能'
   ```
5. **推送到您的分支**
   ```bash
   git push origin feature/amazing-feature
   ```
6. **创建一个 Pull Request**

### 代码风格

请确保您的代码遵循以下风格指南：

- 使用4个空格进行缩进（不使用制表符）
- 类名使用 PascalCase
- 函数名和变量名使用 snake_case
- 添加适当的注释和文档字符串
- 确保代码通过基本的测试
